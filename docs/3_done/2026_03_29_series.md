
### シリーズ機能

frielen01.mp4
frielen02.mp4
frielen2-01.mp4
frielen2-02.mp4

のようなファイルがある場合、frielenとfrielen2というシリーズを作成する。
この場合は、{title}{episode}.拡張子と、{title}-{episode}.{拡張子}です。

ただし、ユーザーによってファイル名は異なるので、マクロで指定できるようにする。
このマクロは、server側のconfig.yaml内で書き、GoのConfig構造体で参照できるようにする。

### DB構造

シリーズテーブル
- id
- series_name_file
- syobocal_title_id
- syobocal_title_name
- syobocal_title_name_en
- comment
- first_year
- first_month
- first_end_year
- first_end_month
- subtitles
  
id, series_name_fileは先ほどの{title}、countは0。それ以外のデフォルト値はNULL。また、videoテーブルにシリーズへの外部キーを持たせるようにする。


### テーブルとファイルの同期

同期方法は、2026_03_22_file_watcher.mdも参考にする。

- 動画ファイルが追加された場合
    - シリーズを作成＋動画にシリーズ外部キーを持たせる
- 動画ファイルが削除された場合
    - 何もしない
- フォルダの監視を始めた場合
  - シリーズを抽出する


### API

基本的には、fsnotifyで監視対象が変化した場合、フォルダ監視APIが呼び出された場合に同期する。ただし、手動で同期する方法も設けたいので、全てのvideoテーブルからシリーズを再作成するAPIも作成する。
